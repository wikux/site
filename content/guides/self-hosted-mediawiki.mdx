Setting up your first MediaWiki instance is relatively accomplishable for most people, especially if you're using a single server like this tutorial assumes, which you most likely are. The first question you should be asking yourself isn't how, but why. Why self-host your own MediaWiki instance over choosing another wiki-farm?

If you don't go with another wiki farm (e.g., Miraheze WikiOasis, etc.), you get all the benefits of controlling your MediaWiki instance, such as:

* Access to enable and config extensions and skins however you please
    * You can even modify them or create your own, like many wikis have!
* Complete server-side control
* Complete permission to change whatever you want however you want
* No reliance on a wiki-farm and their provider
* ...and countless other benefits

BUT. Some of these benefits can also be considered as drawbacks; for example:

* You are responsible for ALL content on your wiki.
* You must write governing terms of use and a privacy policy yourself (if the stock one does not apply).
* You must solely maintain the server, and are responsible for its up-time.

It's up to you if it's worth it. Typically, independent wikis are where the innovation happens in terms of extensions and skins, as it's far easier to test and deploy new versions when you control the server. The choice is yours.

## What should your server look like?

For this guide I'll be using a DigitalOcean Droplet, their form of a VPS. At a minimum, I recommend 1GB of RAM and whatever comes with it. Depending on the size of your wiki, you may want to upgrade this later.

* Login or create a [Digital Ocean](https://www.digitalocean.com/?refcode=4bec7a43ac62) account.
* Navigate to the droplets section. Sometimes you can find 'Create a Droplet' from the home page.
* Choose a datacenter region, ideally close to you or your users (most people choose the US).
* Choose Ubuntu as your operating system, for this guide I'll be using Ubuntu 24.04 LTS x64.
* Based on your budget, a managed database may be diserable down the road, but we'll be skipping that for now, as it adds significant complexity to our setup.
* Click to create the Droplet. Note that if you sign up using our [referral code](https://www.digitalocean.com/?refcode=4bec7a43ac62) you can get $200 free for the first 60 days.

Configure your SSH keys and connect to the droplet's console. You can also do this via the dashboard if you prefer, but that is often at times laggy.

Once you're in the console you can begin the setup.

## Setting up the environment

This guide covers the following stack:

* Docker
* Stable MediaWiki (fpm)
* NGINX
* MariaDB
* A domain managed under Cloudflare

All of these work nicely together and are heavily used.

### Installing Docker

Let's start by installing Docker itself onto our system. You can check the latest install guide by Docker for Ubuntu [here](https://docs.docker.com/engine/install/ubuntu/), or proceed below:

```bash
sudo apt update
sudo apt install ca-certificates curl
```

```bash
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc
```

```bash
sudo apt update
```

```bash
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

---

Then, check if Docker is running:

```bash
sudo systemctl status docker
```

If Docker is not running, start it manually:

```bash
sudo systemctl start docker
```

### Creating the directory

Let's quickly setup the directory. Here, we're going to put our mounts under `/opt/mywiki`, but you can theoretically choose anywhere.

```bash
mkdir -p /opt/mywiki
cd /opt/mywiki
```

Good start. This is going to be the home of where we store a lot of your wiki's configuration files.

### docker-compose.yml

The very first step to actually getting MediaWiki online with Docker is writing a `docker-compose.yml` file. This file essentially tells Docker what it needs to install and where it should run. Here's a good example of what it should look like:

```yaml
services:
  database:
    image: mariadb:10.11
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: rootpassword
      MYSQL_DATABASE: my_wiki
      MYSQL_USER: wikiuser
      MYSQL_PASSWORD: wikipassword
    volumes:
      - db_data:/var/lib/mysql

  mediawiki:
    image: mediawiki:stable-fpm
    restart: always
    depends_on:
      - database
    environment:
      MW_DB_SERVER: database
      MW_DB_NAME: my_wiki
      MW_DB_USER: wikiuser
      MW_DB_PASS: wikipassword
    volumes:
      - ./images:/var/www/html/images
      - ./LocalSettings.php:/var/www/html/LocalSettings.php
      - ./extensions:/var/www/html/extensions
      - ./skins:/var/www/html/skins
  gateway:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf:ro
    depends_on:
      - mediawiki

volumes:
  db_data:
```

You can change the details, such as the database password, then use `nano docker-compose.yml` to paste it into the file directly.

You can see from the file that we're mounting `LocalSettings.php`, `images`, `extensions`, and `skins`. This is because if we include them directly in the volume we may accidentally overwrite them.

You may also observe the MediaWiki image, currently being `mediawiki:stable-fpm`. This will automatically route to the latest stable version of MediaWiki, using php-fpm. Sometimes this isn't ideal, so you may want to replace it with `mediawiki:1.46-fpm` or another tethered release major.

### nginx.conf

Before we start Docker, let's get the `nginx.conf` file working. As an example:

```nginx
# redirect direct IP access

server {
        listen 80;
        server_name YOUR_SERVERS_IP_ADDRESS_HERE;
        return 301 https://YOURDOMAINHERE.com$request_uri;
}

# redirect HTTP access

server {
        listen 80;
        server_name www.yourdomainhere.com yourdomainhere.com;
        return 302 https://yourdomainhere.com$request_uri;
}

# redirect www to apex

server {
        listen 443 ssl;
        server_name www.yourdomainhere.com;

        ssl_certificate /etc/ssl/certs/cloudflare.pem;
        ssl_certificate_key /etc/ssl/certs/cloudflare.key;

        location / {
                return 302 https://yourdomainhere.com$request_uri;
        }
}

server {
        listen 443 ssl;
        server_name yourdomainhere.com;

        ssl_certificate /etc/ssl/certs/cloudflare.pem;
        ssl_certificate_key /etc/ssl/certs/cloudflare.key;

        root /var/www/html;
        index index.php;

        real_ip_header CF-Connecting-IP;
        set_real_ip_from 173.245.48.0/20;
        set_real_ip_from 103.21.244.0/22;
        set_real_ip_from 103.22.200.0/22;
        set_real_ip_from 103.31.4.0/22;
        set_real_ip_from 141.101.64.0/18;
        set_real_ip_from 108.162.192.0/18;
        set_real_ip_from 190.93.240.0/20;
        set_real_ip_from 188.114.96.0/20;
        set_real_ip_from 197.234.240.0/22;
        set_real_ip_from 198.41.128.0/17;
        set_real_ip_from 162.158.0.0/15;
        set_real_ip_from 104.16.0.0/13;
        set_real_ip_from 104.24.0.0/14;

        client_max_body_size 20M;

        # index.php/Page -> /Page
        location ~ ^/index\.php/(.+)$ {
                return 301 /$1$is_args$args;
        }

        location = /robots.txt {
                alias /var/www/robots.txt;
        }

        location ~ \.php$ {
                include fastcgi_params;
                fastcgi_pass app:9000;
                fastcgi_index index.php;
                fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
                fastcgi_param HTTPS on;
        }

        location ^~ /rest.php {
                include fastcgi_params;
                fastcgi_pass app:9000;
                fastcgi_param SCRIPT_FILENAME $document_root/rest.php;
                fastcgi_param HTTPS on;

                fastcgi_split_path_info ^(/rest\.php)(/.*)$;
                fastcgi_param PATH_INFO $fastcgi_path_info;
                fastcgi_param PATH_TRANSLATED $document_root$fastcgi_path_info;
        }

        location ~* ^/images/.*\.php$ {
                deny all;
        }

        location / {
                try_files $uri $uri/ @rewrite;
        }
}
```
